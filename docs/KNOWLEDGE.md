# 中国期货市场知识

## 交易时段

### 日盘（Day Session）：09:00 – 15:00

| 时段 | 时间 | 说明 |
|------|------|------|
| 第一节 | 09:00 – 10:15 | 开盘集合竞价 08:55-09:00 |
| **休市** | **10:15 – 10:30** | 中间休息 |
| 第二节 | 10:30 – 11:30 | |
| **休市** | **11:30 – 13:30** | 午间休市 |
| 第三节 | 13:30 – 15:00 | 收盘集合竞价 14:55-15:00 |

### 夜盘（Night Session）：产品相关

夜盘并非所有品种都有，有夜盘的品种时间因产品而异：

| 夜盘分组 | 时间 | 典型品种 |
|---------|------|---------|
| **有色金属** | 21:00 – 次日 01:00 | CU（铜）、AL（铝）、ZN（锌）、PB（铅）、SN（锡）、NI（镍） |
| **贵金属** | 21:00 – 次日 02:30 | AU（黄金）、AG（白银） |
| **黑色系** | 21:00 – 23:00 | RB（螺纹钢）、HC（热卷）、I（铁矿石） |
| **能源化工** | 21:00 – 23:00 | SC（原油）、FU（燃料油）、BU（沥青）、MA（甲醇）、TA（PTA） |
| **农产品** | 21:00 – 23:00 | P（棕榈油）、Y（豆油）、M（豆粕）、C（玉米） |
| **CZCE 夜盘** | 21:00 – 23:00 | CF（棉花）、SR（白糖）、RM（菜粕）、OI（菜油）、FG（玻璃）、SA（纯碱） |

> **注意**：部分 CZCE 品种（如苹果 AP、红枣 CJ、尿素 UR、纯碱 SA 等）原本无夜盘，但近年有调整，以交易所公告为准。

### 简易分组（项目中 `config.yaml` 使用）

项目中已定义分组：

```yaml
sessions:
  - "09:00-10:15"
  - "10:30-11:30"
  - "13:30-15:00"
  - "21:00-23:30"
```

需要更精细控制时按产品分组分别配置。

## 交易规则

### 集合竞价

| 阶段 | 时间 | 说明 |
|------|------|------|
| 日盘开盘 | 08:55-09:00 | 前 4 分钟接受申报，最后 1 分钟不可撤单 |
| 日盘收盘 | 14:55-15:00 | 最后 1 分钟不可撤单 |
| 夜盘开盘 | 20:55-21:00 | 同上 |

### 涨跌停板

各品种不同，通常为 ±4% ~ ±10%，连续单边市时会扩板。

### 最小变动价位

- 股指期货：0.2 点
- 商品期货：各品种不同（如 RB 1元/吨，AU 0.02元/克）

## 数据源说明

### AKShare `futures_zh_minute_sina`

- 接口：`ak.futures_zh_minute_sina(symbol="RB2410", period="1")`
- 返回列：datetime, open, high, low, close, volume, hold
- 时间范围：最近 5 个交易日
- 时区：返回北京时间（UTC+8）naive datetime
- 限制：非交易日/休市时段无数据，请求过快会被限流（建议每次间隔 ≥0.5s）

### 时间戳存储规范

- **字段**：`ts_ns` — 纳秒级 Unix 时间戳（int64）
- **基准**：北京时间（UTC+8）
- **转换公式**：
  ```python
  dt = pd.to_datetime(row["datetime"])  # pandas naive Timestamp, 北京时间
  # ⚠️ 不能用 dt.timestamp()！pandas Timestamp.timestamp() 把 naive 当 UTC 处理
  # 必须转 Python datetime 再取 timestamp（Python 正确识别为本地时间）
  ts_ns = int(dt.to_pydatetime().timestamp() * 1e9)  # → Unix 纳秒
  ```
- `datetime.timestamp()` 将 naive datetime 当作**本地时间**（北京=UTC+8）转为 UTC 秒
- `pd.Timestamp.timestamp()` 将 naive datetime 当作 **UTC** 处理，导致 +8h 偏移

> **已修复 (2026-06-07)**：原始代码用 `dt.timestamp()` 导致所有 ts_ns 偏移 +8h（数据已全量重刷）。

## 已知问题

### 旧数据时间戳偏移（已修复）

- 现象：夜盘 00:00-02:30 的数据存为 05:00-07:30，偏移约 +5h
- 原因：旧版本 `datetime.timestamp()` 对 naive datetime 解释方式不同，或 AKShare 早期版本返回了不同时区的时间
- 修复：2026-06-07 全量刷新数据后确认 timestamps 正确
